LEYENDA
═══════════════════════════════════════════════════════════════════════════════

Scope:
  auth         = Rutas públicas (sin autenticación)
  super-admin  = Control plane (SUPER_ADMIN global)
  tenant       = Portal de tenant (autenticado)

Roles:
  PUBLIC      = Sin login requerido
  SUPER_ADMIN = Rol global de administración del sistema
  TENANT_*    = Roles dentro del tenant específico
  RESIDENT    = Rol de residente/ocupante
  ALL ROLES   = Accesible por todos los roles autenticados

* Acceso condicional (RESIDENT solo accede a sus units asignadas)

═══════════════════════════════════════════════════════════════════════════════
PATRONES DE ACCESO
═══════════════════════════════════════════════════════════════════════════════

Public Routes (No guard requerido)
  └─ /login, /signup, /invite, /health

Auth Routes (Guard: JWT + Tenant membership)
  ├─ TenantLayout guard chequea: userId + tenantId + rol
  ├─ SUPER_ADMIN: Redirect a /super-admin
  └─ Tenant roles: Permitido acceso

Super-Admin Routes (Guard: SUPER_ADMIN role)
  ├─ SuperAdminLayout guard chequea: SUPER_ADMIN en memberships
  └─ Si no: Redirect a dashboard o login
